# Stickman Fighter — Phaser Edition

A browser-based platform fighter (Brawlhalla-style) built with **Phaser 3**, supporting both offline play and **online multiplayer up to 4 players** via WebRTC (PeerJS).

## 0. Local Development

### Prerequisites

| Tool | Version |
|------|----------|
| Node.js | ≥ 18 |

### Quick start

```bash
npm install          # install Phaser, PeerJS, esbuild, concurrently
npm start            # auto-builds & serves at http://localhost:3000
```

Then open **http://localhost:3000** in your browser.

**Stop**: Press `Ctrl+C` in terminal.

### Watch mode (development)

`npm start` already watches for file changes and auto-rebuilds the bundle. Just edit & save, then refresh your browser — the latest bundle.js will be loaded.

To manually rebuild without serving:

```bash
npm run build        # one-time bundle (no watch)
```

## 1. Deploy on Vercel (end-to-end)

### Prerequisites

| Tool | Version |
|------|---------|
| Node.js | ≥ 18 |
| Vercel CLI | latest (`npm i -g vercel`) |
| Git | any |

### Step 1 — Prerequisites & Install

Make sure your project has all the required files:

```
├── api/
│   └── signal.js          ← Vercel serverless relay
├── css/
│   └── style.css
├── js/
│   ├── network.js
│   ├── main.js
│   ├── scenes/
│   └── ...
├── dist/                  ← auto-generated by npm run build
├── node_modules/          ← auto-generated by npm install
├── index.html
├── package.json           ← defines build command
└── vercel.json            ← routing config
```

Install dependencies:

```bash
npm install
```

### Step 2 — Build & Test locally

```bash
npm run build            # creates dist/bundle.js
npx serve . --listen 3000 # test at http://localhost:3000
```

Verify the game loads and works offline/online before deploying.

### Step 3 — Git & Deploy to Vercel

```bash
git add .
git commit -m "Initial commit"
git push origin main     # push to GitHub/GitLab

vercel login             # follow browser prompt (GitHub / email)
vercel --prod            # deploys to production
```

Vercel will automatically run `npm install` + `npm run build` (configured in `vercel.json`), then serve your game.

Your game is now live at `https://your-project.vercel.app`.

### Step 4 — (Optional) Custom domain

In the Vercel dashboard → **Domains** → add your domain and follow the DNS instructions.

---

### What gets deployed

| Path | Handled by |
|------|-----------|
| `dist/bundle.js` (bundled code) | Vercel **Static** (CDN-cached) |
| `index.html`, `css/` | Vercel **Static** (CDN-cached) |
| `api/signal.js` | Vercel **Serverless Function** (Node.js 18) |

**Build process**: Vercel runs your `buildCommand` from `vercel.json`:
- Installs npm dependencies
- Runs `npm run build` (esbuild bundles all `.js` files + Phaser + PeerJS into `dist/bundle.js`)
- Serves the result as static files

The `api/signal.js` relay handles room token creation and presence — it is **stateless** (no database). Rooms live in memory for up to 10 minutes per serverless instance.

## 2. Play Online — Room Flow

### Host: create a room

1. Open the game → click **🌐 ONLINE** on the main menu.
2. Click **🌐 CREATE ROOM**.
3. The lobby shows your **room code** (e.g. `SF3K9X2PAB`) and a 📋 copy button.
4. Click 📋 to copy the full invite link:
   ```
   https://your-project.vercel.app/?room=sf3k9x2pab
   ```
5. Send the link to up to **3 friends** via any chat/messaging app.
6. Select the game mode (**1 v 1** or **2 v 2**) with the mode pills.
7. Wait for all players to mark themselves **✓ READY**.
8. Click **⚔ START GAME** to begin.

### Guest: join a room

**Option A — via link (recommended)**
- Click the link received from the host.
- The browser opens the game and connects automatically — no copy-paste needed.

**Option B — manual code**
1. Open the game → **🌐 ONLINE** → **🔗 JOIN WITH CODE**.
2. Type or paste the room code (e.g. `SF3K9X2PAB`).
3. Click **→ JOIN**.

### In the lobby

| Element | Description |
|---------|-------------|
| Player cards | Shows name, team colour, and ready state for each slot. |
| `⇆ TEAM 2 / TEAM 1` button | Switch your team at any time before the game starts. |
| Green dot | Player is ready. |
| Mode pills | Host-only: switch between **1 v 1** and **2 v 2**. |
| **✓ READY** | Toggle your ready state. Guests must ready-up before host can start. |
| **⚔ START GAME** | Host-only, enabled when ≥ 2 players and all non-host players are ready. |

### After a match

- **REMATCH** — returns everyone to the lobby (connection stays live).
- **MENU** — host disconnects; all clients are kicked back to the main menu.

## 3. Game Modes

| Mode | Players | Description |
|------|---------|-------------|
| **1 v 1 — 2 Players** | 2 local | Both players share one keyboard |
| **1 v 1 — VS AI** | 1 local | Fight against a bot |
| **2 v 2 — Team Battle** | 2 local + 2 bots | Team 1 (you + AI ally) vs Team 2 |
| **Tournament** | 1 local | Single-elimination bracket vs AI opponents |
| **Online 1 v 1** | 2 online | First to knock out 3 stocks wins |
| **Online 2 v 2** | 2–4 online | Teams of 2; last team with stocks wins |

> **Stock system**: each player starts with **3 lives**. You lose a stock when knocked beyond the blast zones. Last player/team standing wins.

## 5. Combat System

### Damage meter

Damage is displayed as a **percentage (0% – 200%+)** below each character's name, Brawlhalla-style. The higher your damage, the farther you fly when hit.

- **0–79%** — cyan/red (normal)
- **80–149%** — orange (high)
- **150%+** — flashing red (critical)

### Attack types

#### Light attacks (instant startup)

| Input | Ground | Air |
|-------|--------|-----|
| Neutral | 3-hit punch combo | 3-kick aerial combo |
| Forward | Sliding kick (stronger knockback) | — |
| Down | Soccer slide-tackle (horizontal pop) | Diagonal dive-kick |

#### Heavy attacks (70 ms windup — telegraphed)

| Input | Ground | Air |
|-------|--------|-----|
| Neutral | Power punch | Rising uppercut (launches up) |
| Forward | Dash double-leg kick | — |
| Down | Upward crouching kick (sends enemy up) | Straight power-dive |

> **How to directional attack**: hold a direction and press Light or Heavy simultaneously.

### Dodge & Dash

| Technique | How | Effect |
|-----------|-----|--------|
| **Ground Dash** | Double-tap `A` or `D` (or hold Dodge on ground) | 200 ms burst of speed, 500 ms momentum carry |
| **Air Dodge** | Dodge button in the air | 1-second invincibility window, 90 px micro-dash |
| **Fast-fall** | Hold `S`/`↓` while airborne | 1.5× gravity multiplier — reach the ground faster |

Dodge has a **3-second cooldown** from the button press, so use it wisely.

### Knockback & launching

- Knockback force scales with the target's current damage %.
- **Horizontal blasts**: left/right blast zones at x = −300 and x = 1580.
- **Vertical blast zone**: bottom at y = 900, top at y = −750.
- After losing a stock, you respawn with **2 seconds of invincibility**.

### Energy bar

- Gain energy by **landing hits** (+8 per hit) or **being hit** (+5 per hit).
- When the bar is **full (100)** it pulses gold.
- Press Light + Heavy simultaneously to fire your **Ultimate**.
- Using the Ultimate consumes all energy.

## 6. Skill Items & Ultimates

Glowing orbs spawn on floating platforms every ~14 seconds (max 3 active at once). Walk into one to collect it — the orb replaces your default Ultimate until used.

| Icon colour | Skill | Ultimate Effect |
|-------------|-------|----------------|
| 🟠 Orange | **FIRE** | Rocket punch — dash through enemy at high speed (fast startup, 28 dmg) |
| 🟡 Yellow | **THUNDER** | Leap + two-fist overhead slam (30 dmg, launches upward) |
| 🟣 Purple | **VOID** | Wide radial shockwave — hits ALL enemies in 240 px range (32 dmg) |
| 🔴 Red | **BERSERK** | 5-hit rapid combo, each hit holds enemy close; final hit launches (total ~48 dmg) |

> Skill items despawn after **22 seconds** if uncollected. Killing a player with high damage while they hold a skill does not drop the orb.

## 8. Tech Architecture

```
Browser A (host)             Browser B/C/D (clients)
┌──────────────────┐         ┌──────────────────┐
│  GameScene       │         │  GameScene        │
│  runs full       │◄───────►│  input → host     │
│  physics         │  WebRTC │  apply snapshots  │
│  broadcasts      │  P2P    │  render locally   │
│  state @30Hz     │         │                   │
└──────────────────┘         └──────────────────┘
         │                            │
         └──────────┬─────────────────┘
                    │
            ┌───────▼────────┐
            │  PeerJS cloud  │
            │  (STUN/ICE)    │
            └───────┬────────┘
                    │ room token only
            ┌───────▼────────┐
            │ api/signal.js  │
            │ Vercel Function│
            │ (no DB, no     │
            │  game state)   │
            └────────────────┘
```

- **Host-authoritative**: host runs full physics, broadcasts fighter snapshots to all clients at ~30 Hz.
- **Client prediction**: clients run local physics for their own fighter and apply server correction if drift exceeds 80 px.
- **Input relay**: clients send keyboard state to host on every change (not every frame).
- **Signaling**: the Vercel serverless function only stores room tokens in memory — it never sees game data.
- **No database**: fully stateless. If the Vercel instance cold-starts, the room list resets; the PeerJS connection is unaffected as long as both peers are connected.

### Browser requirements

- Chrome 90+ / Firefox 88+ / Edge 90+ / Safari 15+
- WebRTC must not be blocked by a corporate firewall/proxy.
- On restricted networks (hotels, some corporate VPNs) WebRTC may fail — a TURN server would be needed for 100% reliability.
